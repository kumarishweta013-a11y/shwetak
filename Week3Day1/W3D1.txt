CSS Selectors [Cascading style sheet]: 
-------------------------------------
CSS is used for style and design a web page
CSS selectors are fast because browsers understand them natively and can locate elements quickly.

1. By ID :
Syntax : #idValue 
Example : #username

2. By Class :
Syntax : .classValue
Example : .inputLogin

3. By Attribute :
[tagName[attribute='value']]
Example : input[id='username']

4. Descendant Combinator 
Example : form input

5. Child Combinator
Example : form > input

6. Immediate Siblings
Example: label + input

7. Descendant Siblings
Example: label ~ input

Attribute priority :
-----------------------
1️⃣ aria-label - Accessibility-based, stable, first choice
2️⃣ id - Unique, fast
3️⃣ name -Usually stable, not always unique
4️⃣ class - For styling, changes often → least reliable

Playwright locator methods :
----------------------------

nth(), first(), and last() are Playwright Locator API methods.

They help you choose which element to act on when a locator matches multiple elements.

Strict Mode Violation Error :
-----------------------------
When a locator matches more than one element, Playwright throws an error to prevent ambiguity.
To fix this, use nth(), first(), or last() to specify which element you want to interact with.

Dropdowns:
----------
A dropdown (or select menu) is a UI element that allows the user to choose one or more options 
from a list.
There are two main types of dropdowns in web applications:


1. Standard / Native Dropdown
-----------------------------

A Standard (or Native) Dropdown is a dropdown list created using the HTML <select> tag and its 
child <option> elements.
It is called native because it's a built-in browser control — handled directly by the browser's 
rendering engine, not by JavaScript or CSS styling.

Ex:<select id="country">
  <option value="IN">India</option>
  <option value="US">United States</option>
  <option value="UK">United Kingdom</option>
</select>

how to handle using build-in methods
------------------------------------

1) await page.locator('#country').selectOption('IN');
or
2) await page.selectOption('#country', 'IN');

Key Features:
------------

Uses native HTML elements (<select>, <option>).
Fully supported by browsers (no JavaScript required).
Easy to automate with Playwright's selectOption().
Supports value, label, and index attributes.

1) By value attribute:
-------------------
The value is the hidden attribute inside the <option> tag.
Playwright uses this when you provide a string.

await page.locator('#country').selectOption({value:'US'});

this seletcs:
<option value="US">United States</option>

2)By label (visible text)
-------------------------

The label refers to the display text that users see in the dropdown.

await page.locator('#country').selectOption({ label: 'United Kingdom' });

this selects:
<option value="UK">United Kingdom</option>

3) By index (position in the list)
----------------------------------

The index represents the zero-based position of the option.

await page.locator('#country').selectOption({ index: 0 });

this selects:
<option value="IN">India</option>

priority among the 3 options
----------------------------
Ex:


Value → loginBtn (value of the attribute data-testid)
Label / Visible text → Sign In

Even if text changes from Sign In to Login,
the locator still works.
Value comes first because it is stable, unique, and reliable.

Priority1: Value
Priority2: label/visible text
Priority3: index

why value is given first priority
---------------------------------
this selects:
<option value="US">United States</option>

In a standard HTML form, the <option>'s value is what actually gets sent to the server when a 
form is submitted — not the visible text.

If a user selects “United States”,
the browser actually sends → country=US
